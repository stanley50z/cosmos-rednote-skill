# 封面文字 (Cover Text)

How the text that sits **on** the cover image is written. Core deliverable of the skill — not the cover image itself. The image is the user's own photo; we generate the words that land on it.

> Sourced from the Cosmosworks 小红书工作流 V2, Section 02 · 标题+封面文字, Section B · 货架感, Section J · 避坑.

## The one rule that matters most

> **封面 ＝ 标题，但不重复** —— 是同一件事，但透露不同信息点。

| | 标题 (title) | 封面文字 (cover text) |
| --- | --- | --- |
| 落点 | 正文上方，给搜索/列表看 | 落在封面图上，给信息流第一眼看 |
| 语气 | 更正式 · 客观 | 偏情绪 · 网感 · 口语 |
| 承担 | "这是什么内容" + 被搜到 | "为什么我要点开" |

**Worked example (from the doc):**

```
标题：骑了一年 Vapor SL：3千公里真实长测
封面主文字：谁懂啊
封面副文字：3千公里没进过店
```

Same fact (3000 km, one year). Title states it; cover text reacts to it. If your cover text is just the title with fewer characters, you failed this rule.

## 主文字 + 副文字 structure

Cover text is **two fields**:

- **主文字 (main)** — the emotional hook. Biggest type on the cover. **≤10 Chinese characters.** One feeling or one bold claim. This is the "谁懂啊 / 地铁戒断成功 / 爬坡不喘了" line.
- **副文字 (sub)** — the specific payoff or proof. Smaller, near the main line. **≤12 Chinese characters.** One concrete number/detail: "3千公里没进过店 / 通勤 12km 再没堵过 / 坡道实测 18%".

Main earns the glance; sub earns the tap. Don't merge into one long line — the size contrast between them is what reads as a poster instead of a caption.

## Character limits (hard)

- 主文字 ≤ 10 字 — one core information point. Large font, contrasting color.
- 副文字 ≤ 12 字 — one specific payoff or searchable detail.
- Over the limit ⇒ stops reading as a headline, starts reading as a sentence. **Cut, don't shrink.**

## SEO: hook and keyword are the same word

Rednote indexes text **embedded inside the cover image** for search relevance. So:

- Put **one searchable keyword** inside the cover text (in 主文字 or 副文字).
- Make the emotional hook and the SEO keyword the **same word** where possible. Fold it in — don't bolt it on as dead weight.
- For Cosmosworks that keyword is usually the product/use-case noun: `通勤`, `爬坡`, `长测`, `电助力`, a model name.

## Tone: 平视感 (peer, not teacher)

> 叙事立场从"我来教你"变成"我也在练"——平视感是最大的情感连接。

Avoid the instructional / lecturing tone in both title and cover:

- ✗ "教你如何选电助力" → ✓ "我也纠结了三个月才下手"
- ✗ "新手必看的 5 个要点" → ✓ "新手踩过的坑我替你踩了"

Peer voice over expert voice. Single biggest emotional connector; easiest thing to get wrong.

## Emoji: 过气, cut them

> emoji 过气了，不合时宜地放一堆 emoji 真的是灾难。会分段比会放 emoji 重要得多。

- **0 emoji preferred.** Max 1, only if it genuinely earns its place.

## 违禁词 (banned words)

Flag and remove all absolute superlatives and regulated claims:

- Hard-banned by platform: `最` / `第一` / `唯一` / `国家级` / `顶级` / `首个` and similar absolutes.
- Brand-risk for e-bike: medical/safety guarantees, range/speed claims stated as absolute fact.

If a line needs a superlative to work, the line is weak — rewrite with a concrete number instead (`最快` → `比上一代快 1.8 秒`).

## 真实感 ＞ 精美感

> 反例：儿科笔记用精美卡通图反而像营销内容。正例：改成挂号单照片反而更真实、点击更高。

The words should sound like a real person typed them, not ad copy. An over-designed cover reads as marketing and gets scrolled past.

## 货架感 / the shelf test

> 尺寸：竖图 3:4（占双列流最大面积）。
> 测试法：把封面截原图扔到目标受众的真实信息流里跟周围 3 条对比。

Imagine the cover dropped into the real double-column feed next to three neighbors. If it doesn't out-pull them at thumbnail size, the text is wrong.

## 9 cover paradigms (封面范式)

| 范式 | 特征 | 适用场景 | 需要照片? |
| --- | --- | --- | --- |
| 大字报型 | 几个超大字 + 撞色底 | 知识干货 / 观点 | 否（纯色/弱底图即可） |
| 多图拼贴型 | 同主题 2–9 张拼贴 | 旅行 / 美食打卡 / 合集 | 是（多张） |
| 人像抠图型 | 真人 / IP 抠出 | 人格化账号 / 穿搭 | 是 |
| 推荐集锦型 | 纯文字升级版 | 知识类需要图片辅助 | 否 |
| 经典图文型 | 大图 + 花字 | 适用面最广 | 是 |
| 留白排版型 | 简单朴素 + 留白 | 摄影 / 设计 / 生活美学 | 弱 |
| 极简摄影型 | 高级感照片 + 少量文字 | 摄影 / 产品 | 是（必须） |
| 泥石流型 | 故意做粗糙，大巧若拙 | UGC 氛围 / 书摘 | 否 |
| 表情包/素人感型 | 反精致、幽默 | 涨粉初期 | 否 |

Two 顶层打法 sit above the table:
1. **悬念出题型** — 封面只给问题/景象不给答案，标题补诱惑。
2. **结构化大而全型** — 首屏告知所有信息点（"8个秘密"/"10大副业"），但让人觉得要点进去仔细看。

If user has no photo for a photo-required paradigm, switch to a no-photo paradigm (大字报 / 推荐集锦 / 泥石流 / 表情包).

## Self-check before delivery

- [ ] **可逆性测试** — 遮住封面只看标题、遮住标题只看封面，**两边都让人想点**；两条读起来是"补充"不是"复读"。
- [ ] 封面 ≠ 标题 — different sentence, same fact, different info point.
- [ ] 主文字 ≤ 10 字, 副文字 ≤ 12 字.
- [ ] Size contrast between 主 and 副 is obvious.
- [ ] One searchable keyword lives inside the cover text.
- [ ] Peer tone — no "我来教你".
- [ ] 0 (max 1) emoji.
- [ ] No 违禁词 (最/第一/唯一/…).
- [ ] Passes the shelf test against 3 neighbors at thumbnail size.
